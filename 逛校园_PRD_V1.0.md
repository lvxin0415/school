## 1. 项目概述

### 1.1 背景与目标

逛校园是一款面向在校大学生及校园访客的移动应用，聚焦三大核心场景：**匿名留言树洞、学习打卡激励、3D 沉浸式校园参观**。通过 uni-app + uniCloud 实现快速全端交付，上线 iOS / Android 双平台。

### 1.2 目标用户

| 角色        | 典型用户            | 核心诉求                      | 主要使用功能              |
| ----------- | ------------------- | ----------------------------- | ------------------------- |
| 在校生      | 小黄，18 岁大学生   | 吐槽/互动、记录成长、了解校园 | 留言墙、学习记录、3D 参观 |
| 教职工      | 小颖，28 岁大学老师 | 发布活动通知、了解学生动态    | 消息、活动公告            |
| 毕业生/访客 | 小薛，22 岁毕业生   | 回顾校园、沉浸式漫游          | 3D 校园参观、留言墙       |
| 管理员      | 系统管理人员        | 内容审核、素材管理、用户管理  | 管理后台（App 内）        |

### 1.3 核心功能全览

| --------------- | ------ | ------------------------------------------- |
| 账号注册 / 登录 | P0     | 账号密码注册登录，管理员登录自动跳转后台    |

## 2. 技术架构

### 2.1 技术栈总览

| 层级      | 技术选型                           | 说明                              |
| --------- | ---------------------------------- | --------------------------------- |
| 前端框架  | uni-app（Vue 3 + Composition API） | 一套代码编译 iOS / Android        |
| UI 组件库 | uView UI 2.x + uni-ui              | 基于紫色主题定制                  |
| 3D 渲染   | three.js + FBXLoader（renderjs）   | 加载 .fbx 模型，手势交互          |
| 状态管理  | Pinia                              | 全局用户信息、管理员标识          |
| 后端服务  | uniCloud 云函数                    | Serverless，免运维                |
| 数据库    | uniCloud 云数据库（MongoDB）       | JSON 文档型，JQL 查询             |
| 文件存储  | uniCloud 云存储 + CDN              | 存储头像、帖子图片、.fbx 模型     |
| 用户体系  | uni-id 内置账号系统                | 支持 role 字段区分普通用户/管理员 |
| 内容安全  | uniCloud 内容安全插件              | 帖子文字 + 图片自动审核           |

### 2.2 角色权限设计

uni-id 用户文档新增 `role` 字段，取值 `user`（普通用户）或 `admin`（管理员）。系统初始化时预置一个管理员账号（username: admin，密码加密存储）。登录时云函数检测 `role` 字段，前端据此决定跳转路径。

| 角色     | role 字段值 | 说明                                         |
| -------- | ----------- | -------------------------------------------- |
| 普通用户 | `user`      | 注册后默认角色，只能访问用户端功能           |
| 管理员   | `admin`     | 登录成功后自动跳转管理后台首页，拥有全部权限 |

> 💡 初始管理员账号由开发人员在 uniCloud 云数据库中直接插入，不对外开放注册。

### 2.3 云数据库集合设计

| 集合名称        | 主要字段                                                                        | 说明                                          |
| --------------- | ------------------------------------------------------------------------------- | --------------------------------------------- |
| `uni-id-users`  | \_id, username, password, role, avatar, nickname, created_at                    | 用户表，role='admin' 为管理员                 |
| `message_wall`  | \_id, content, images, likes, comments, status, created_at                      | 留言墙帖子，status: pending/approved/rejected |
| `mw_comments`   | \_id, post_id, content, created_at                                              | 留言墙评论（完全匿名，无 user_id）            |
| `study_records` | \_id, user_id, date, duration_min, subject, note, streak_days                   | 学习打卡记录                                  |
| `badges`        | \_id, user_id, badge_type, earned_at                                            | 用户已获得勋章                                |
| `campus_models` | \_id, name, file_url, thumbnail_url, description, hotspots, status, uploaded_at | 3D 校园模型（管理员上传）                     |
| `activities`    | \_id, title, cover_url, content, start_time, end_time, location, created_by     | 活动/公告（管理员发布）                       |
| `notifications` | \_id, user_id, type, content, read, created_at                                  | 系统通知                                      |
| `feedback`      | \_id, user_id, content, images, reply, status, created_at                       | 用户意见反馈                                  |

### 2.4 3D 模型技术方案

3D 校园参观采用 three.js 在 uni-app **renderjs** 环境内渲染 `.fbx` 格式模型：

- 管理员在 App 后台上传 `.fbx` 文件至 uniCloud 云存储，获取 CDN URL 写入 `campus_models` 集合
- 用户端从云数据库查询最新启用的模型 URL，通过 renderjs 动态加载 three.js + FBXLoader
- **OrbitControls** 实现手势交互：单指拖动旋转、双指捏合缩放、双击重置视角
- 模型关键位置可配置 **Sprite 热点标注**，点击弹出简介卡片
- 加载期间显示紫色进度条动画；低端机自动降级为静态全景图

> 💡 模型文件建议压缩至 30MB 以内，Blender 导出时勾选 Apply Transform 并烘焙纹理。

## 3. 用户端功能需求

### 3.1 登录 / 注册

#### 功能说明

- **账号注册**：填写用户名 + 密码（二次确认）+ 昵称，提交后写入 uni-id-users，role 默认 `user`
- **账号登录**：输入用户名 + 密码，云函数校验，返回 Token 存本地
- **管理员登录**：同一登录页，云函数检测 role='admin' 后返回特殊标识，前端自动跳转管理后台首页
- **忘记密码**：暂不支持自助找回，由管理员在后台重置
- **自动登录**：Token 有效期 30 天，每次启动静默校验，有效则免登录
- **退出登录**：清除本地 Token，返回登录页

#### 页面流程

1. 首次启动 → 3 屏引导页（留言墙 / 学习记录 / 3D 参观）→ 登录页
2. 登录成功 → 检测 role：`admin` → 管理后台首页；`user` → 用户端首页
3. 注册页：输入用户名、密码、昵称 → 提交 → 登录成功 → 用户端首页

---

### 3.2 首页

- 顶部搜索框 + 消息通知图标（未读数角标）
- **Banner 轮播**：展示管理员发布的活动 / 公告（最多 5 条）
- **功能宫格**（2 行 × 4 列）：留言墙、学习记录、校园参观、活动广场、课程表、成绩、心理健康、设置
- **推荐内容流**：最新审核通过的留言墙帖子 + 近期活动，无限下滑分页
- 下拉刷新内容流

---

### 3.3 留言墙（匿名树洞 / BBS）— 核心功能

#### 功能概述

留言墙采用**完全匿名**设计，用户发帖、评论均不显示真实身份，仅展示随机生成的匿名昵称（如「匿名小熊」）。帖子需经管理员审核后公开展示。

#### 功能列表

- **发布帖子**：文字（500 字以内）+ 图片（最多 9 张）+ 位置标签 + 话题标签（#）
- **匿名身份**：每条帖子随机分配匿名昵称 + 随机头像色块，不关联用户真实账号
- **审核机制**：帖子提交后状态为 `pending`，管理员审核通过后 `status` 改为 `approved` 方可展示；内容安全 API 自动审核通过则直接 `approved`
- **帖子列表**：时间倒序瀑布流，无限下滑分页，显示文字摘要 + 首图 + 点赞数 + 评论数
- **帖子详情**：完整内容、图片大图预览、评论区
- **评论**：同样匿名，支持二级回复，楼层编号展示
- **点赞**：匿名点赞，无需登录状态关联
- **话题标签**：# 话题聚合页，展示同话题所有帖子
- **搜索**：按关键词搜索已审核通过的帖子
- **举报**：对帖子/评论发起举报，通知管理员处理
- **我的帖子**：个人中心查看自己历史发帖（含审核状态）

#### 主要云函数

| 云函数       | 触发场景     | 主要逻辑                                                           |
| ------------ | ------------ | ------------------------------------------------------------------ |
| `createPost` | 发布帖子     | 图片上传云存储，内容安全审核，写入 message_wall（status: pending） |
| `getPosts`   | 获取帖子列表 | 只查询 status='approved'，分页，关联评论数                         |
| `likePost`   | 点赞 / 取消  | 原子操作更新 likes 字段                                            |
| `addComment` | 发评论       | 写入 mw_comments，触发帖主通知（仍匿名）                           |

### 3.4 学习记录 — 核心功能

#### 功能概述

帮助用户养成学习习惯，通过打卡计时、可视化统计、排行榜、积分与勋章体系形成完整激励闭环。

#### 打卡计时

- 进入学习记录页，点击「开始学习」启动计时器（App 切后台暂停计时并提示）
- 计时结束后填写：科目标签（可自定义）+ 打卡感言（选填）+ 配图（选填）
- 点击「完成打卡」触发庆祝动效，数据写入 `study_records`
- 支持手动输入时长（无需计时器），适合补录
- 每日允许多次打卡，同一天累计计算总时长

#### 学习统计图表

- **学习日历**：月历视图，有打卡的日期高亮（按时长深浅渐变）
- **周统计**：柱状图展示本周每天学习时长
- **科目占比**：饼图展示各科目学习时间分布
- **累计数据**：总学习时长、总打卡天数、最长连续打卡天数

#### 排行榜

- **全校榜**：按当周学习总时长排名前 50，展示匿名昵称 + 时长 + 连续天数
- 每周一 00:00 自动重置周榜，历史周榜存档可查
- 排行榜仅展示时长，不关联用户真实身份

#### 积分 / 勋章体系

| 触发行为                   | 获得积分 | 备注             |
| -------------------------- | -------- | ---------------- |
| 每日首次打卡               | +10      | 每天限一次       |
| 打卡时长 ≥ 1 小时          | +5       | 可与首次打卡叠加 |
| 连续打卡 7 天              | +30      | 达成即发放       |
| 连续打卡 30 天             | +100     | 达成即发放       |
| 发布留言墙帖子（审核通过） | +5       | 每天最多 3 次    |

**勋章列表（达成条件自动解锁）：**

| 勋章 | 名称     | 解锁条件                         |
| ---- | -------- | -------------------------------- |
| 🌱   | 萌新     | 完成首次打卡                     |
| 🔥   | 学习达人 | 累计打卡 30 天                   |
| ⭐   | 坚持不懈 | 连续打卡 7 天                    |
| 👑   | 卷王     | 单周学习时长排行榜第 1 名        |
| 📝   | 话痨     | 留言墙发帖累计 20 条（审核通过） |

### 3.5 3D 校园参观 — 核心功能

#### 功能概述

从云端加载管理员上传的 `.fbx` 3D 校园模型，在 App 内渲染展示，用户可通过手势自由旋转视角、缩放远近，点击建筑热点查看简介。

#### 功能列表

- **模型展示**：加载最新启用的 .fbx 模型，带环境光 + 方向光渲染
- **旋转**：单指拖动旋转 360° 水平视角
- **缩放**：双指捏合放大/缩小，限定最大/最小距离防止穿模
- **重置**：双击屏幕恢复默认视角
- **热点标注**：管理员配置的建筑标注点，3D 空间中悬浮显示图标
- **热点详情**：点击标注弹出底部卡片，展示建筑名称 + 功能简介 + 图片
- **加载进度**：显示紫色主题进度条 + 加载百分比
- **降级方案**：检测到 WebGL 不支持时，展示静态 360° 全景图替代
- **无模型时**：展示占位提示「校园模型准备中，敬请期待」

## 4. 管理员后台功能需求

### 4.1 入口与导航

管理员使用预置账号（`admin`）登录，登录云函数检测 `role='admin'` 后返回 `isAdmin: true`，前端跳转至独立的管理后台首页（`/pages/admin/index`）。管理后台有独立底部导航栏，与用户端完全隔离。

**管理后台底部导航（5 个 Tab）：**

| Tab  | 图标                  | 说明                |
| ---- | --------------------- | ------------------- |
| 概览 | bar-chart-outline     | 数据大盘 + 快捷入口 |
| 内容 | document-text-outline | 帖子审核 + 内容管理 |
| 模型 | cube-outline          | 3D 模型上传与管理   |
| 用户 | people-outline        | 用户列表与操作      |
| 发布 | megaphone-outline     | 活动 / 公告发布管理 |

---

### 4.2 数据概览

- 今日新增用户数、今日新增帖子数、今日全校总学习时长
- **待审核帖子数**（红点角标提醒）
- 近 7 日用户活跃趋势折线图
- 快捷入口：直达帖子审核、模型管理

---

### 4.3 留言墙内容管理

#### 帖子审核

- **待审核列表**：按时间倒序展示所有 `status='pending'` 的帖子，显示文字内容 + 图片缩略图
- **审核操作**：通过（`status→approved`）/ 拒绝（`status→rejected`，可填写拒绝理由）
- **批量操作**：全选 + 批量通过 / 批量拒绝
- **自动审核**：内容安全 API 检测无风险则自动 `approved`，风险内容转人工审核队列

#### 已发布内容管理

- 所有已过审帖子列表，支持按时间/点赞数/评论数排序
- 删除帖子（同步删除关联评论）
- 查看举报：展示被举报帖子 + 举报原因，决定保留或删除
- 关键词搜索帖子

---

### 4.4 3D 模型管理 — 核心管理功能

#### 模型列表

- 展示所有上传的 3D 模型（名称、缩略图、文件大小、上传时间、状态）
- 状态：**启用**（用户端展示此模型）/ **停用**（隐藏）
- 同一时间只能有一个模型处于「启用」状态，切换时自动停用原有模型

#### 上传新模型

- 从手机本地选择 `.fbx` 文件
- 填写模型名称、描述、封面缩略图
- 上传至 uniCloud 云存储，进度条展示上传进度
- 上传完成后写入 `campus_models` 集合，初始状态为「停用」，由管理员手动启用

#### 热点标注管理

- 为指定模型添加热点标注：输入建筑名称、简介文字、展示图片、3D 坐标（X/Y/Z）
- 3D 坐标可手动填写（从 Blender 导出时记录）
- 支持编辑 / 删除已有标注

#### 删除模型

- 删除模型记录及云存储中的对应文件
- 启用中的模型不允许直接删除，需先切换到其他模型或停用

---

### 4.5 用户管理

- **用户列表**：展示所有注册用户，显示用户名、昵称、注册时间、状态、积分
- **搜索用户**：按用户名 / 昵称搜索
- **封禁用户**：封禁后该账号无法登录，帖子自动隐藏，封禁原因记录
- **解封用户**：恢复账号正常状态
- **重置密码**：将用户密码重置为随机密码，通过 App 内消息通知用户（用户登录后必须修改密码）
- **查看用户详情**：发帖记录、学习记录、积分明细

---

### 4.6 活动 / 公告发布

#### 发布活动

- 填写活动标题、封面图、详情富文本内容、开始时间、结束时间、活动地点
- 发布后展示在活动广场及首页 Banner（Banner 取最新 3 条活动）
- 支持置顶：最多 1 条置顶公告

#### 活动管理

- 活动列表：展示所有活动，支持编辑、下架、删除
- 下架：活动从用户端隐藏，数据保留
- **发送系统通知**：向全体用户推送指定消息

## 7. 非功能性需求

### 7.1 性能

- 冷启动时间 < 3s（中端机，4G 网络）
- 页面切换帧率 ≥ 50fps
- 3D 模型首屏加载 < 8s（WiFi 环境，模型文件 ≤ 30MB）
- 接口响应 P95 < 800ms
- 图片懒加载 + CDN 加速，列表帖子缩略图压缩至 200KB 以内

### 7.2 安全性

- 所有云函数调用需携带有效 uni-id Token，无 Token 或过期直接返回 401
- 管理员权限云函数额外校验 `role='admin'`，普通用户调用直接拒绝
- 用户密码使用 bcrypt 加密存储，禁止明文
- 帖子图片经内容安全 API 检测，违规图片拦截不入库
- 同一账号每分钟发帖 ≤ 3 次（防刷）
- 管理员账号初始密码要求：≥ 10 位，含大小写字母 + 数字

### 7.3 可用性与兼容性

- 支持 iOS 13+ / Android 8.0+
- 适配刘海屏、全面屏底部安全区域（uni-app safe-area-inset 处理）
- 弱网（2G/3G）下核心功能可用，图片加载失败显示占位图
- 3D 参观页 WebGL 不支持时自动降级，不崩溃
